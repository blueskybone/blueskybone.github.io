This task is meant to be solved with list comprehension.

Write a function headOrLast :: [String] -> Char -> [String] that, given a list of strings and a character, evaluates to a list with all the strings of the input list that either begin or end with the input character.

example:

headOrLast ["hello","old","friends"] 'o' result: ["hello", "old"]

headOrLast :: [String] -> Char -> [String]
headOrLast xs c = [ x | x <- xs , head x == c || last x == c ]

2.

This task is meant to be solved with recursion.

Write a function nextIsSmaller :: [Int] -> [Int] that, given a list of numbers, produces a list with all elements of the input list such that the element is followed by a smaller number in the input list (the next number is greater).

The numbers need to be in the same order relative to each other in the output list as they are in the input list.

example:

function call: nextIsGreater [0,5,2,3,2,2,3,1] result: [5,3,3]

nextIsSmaller :: [Int] -> [Int]
nextIsSmaller  [] = []
nextIsSmaller  [x] = []
nextIsSmaller (x:y:xy) = [y | x > y] ++ nextIsSmaller(y:xy)

A few more recursive functions

Now that we know how to generally think recursively, let's implement a few functions using recursion. First off, we'll implement replicate. replicate takes an Int and some element and returns a list that has several repetitions of the same element. For instance, replicate 3 5 returns [5,5,5]. Let's think about the edge condition. My guess is that the edge condition is 0 or less. If we try to replicate something zero times, it should return an empty list. Also for negative numbers, because it doesn't really make sense.

replicate' :: (Num i, Ord i) => i -> a -> [a]  
replicate' n x  
    | n <= 0    = []  
    | otherwise = x:replicate' (n-1) x

Next up, we'll implement take. It takes a certain number of elements from a list. For instance, take 3 [5,4,3,2,1] will return [5,4,3]. If we try to take 0 or less elements from a list, we get an empty list. Also if we try to take anything from an empty list, we get an empty list. Notice that those are two edge conditions right there. So let's write that out:

take' :: (Num i, Ord i) => i -> [a] -> [a] 
take' n _ 
   | n <= 0  = [] 
 take' _ []   = [] 
 take' n (x:xs) = x : take' (n-1) xs

Function composition

In mathematics, function composition is defined like this: (f . g)(x) = f(g(x)) , meaning that composing two functions produces a new function that, when called with a parameter, say, x is the equivalent of calling g with the parameter x and then calling the f with that result.

In Haskell, function composition is pretty much the same thing. We do function composition with the . function, which is defined like so:

(.) :: (b -> c) -> (a -> b) -> a -> c 

f . g = \x -> f (g x)

If you get three values 6, then you get 100 points

Otherwise, if you get three times the same value, you get 3*(sum of dice values), e.g. three fives gives you 3*(5+5+5) = 45

Write a function that, given three dice values (Ints), calculates the points.

This task is suitable for pattern matching and guards.

-- Task3
getPoints :: Int -> Int -> Int -> Int
getPoints 6 6 6 = 100 
getPoints x y z
 |x == y && y == z = x * 3 + y * 3 + z * 3
 |x == y && y /= z = x * 2 + y * 2 + x
 |x == z && y /= z = x * 2 + y + z * 2
 |x /= y && y == z = x + y * 2 + z * 2
 |otherwise = sum
 where sum = x + y + z

We say that character pair (c1,c2) appears in string s with gap g, if c1 is before c2 and there are exactly g characters between c1 and c2 in s.

Write a function gap :: (Char, Char) -> Int -> String -> Int that, given a pair (c1,c2), a gap g and a string s returns an Int telling how many times (c1,c2) appear in s with gap g.

For example:

function call: gap ('a','b') 1 "aaabbb" result: 2

 -- Task4
gap :: (Char, Char) -> Int -> String -> Int
gap (c1, c2) g [] = 0
gap (c1, c2) g [x] = 0
gap (c1, c2) g (x:xs)
 |(length xs) <= g = 0
 | x == c1 && (xs !! g) == c2 = 1 + gap (c1, c2) g (xs)
 | otherwise = 0 + gap (c1, c2) g (xs)

3.IntDistCalculable typeclass membership needs a function for calculating an integer distance between two values. Make Bool, Integer, Double and Maybe a instances of IntDistCalculable in the following way:

For Bool, the distance for different values is 1 and for the same values it is 0. E.g. integerDist True True returns 0.

For Integer, the distance is the absolute value of the difference. E.g. integerDist (-10) 10 returns 20.

For Double, the distance is the absolute value of the rounded difference. E.g. integerDist 1.4 2.5 returns 1.

For Maybe a, the type variable a is required to be an instance of IntDistCalculable and the distance is calculated using the implementation of integerDist for it. If one or both of the values are Nothing then the result is 0. E.g. integerDist (Just True) (Just False) returns 1. The definition for IntDistCalculable is given below: class IntDistCalculable a where integerDist :: a -> a -> Intege

-- Task3

class IntDistCalculable a where
   integerDist :: a -> a -> Integer
   
instance IntDistCalculable Bool where
   integerDist a b
       | a == b = 1
       | a /= b = 0 

instance IntDistCalculable Integer where
   integerDist a b
       | a >= b = a - b
       | a < b = b - a

instance IntDistCalculable Double where
   integerDist a b
       | a >= b = round ( a - b )
       | a < b = round ( b - a )    

instance IntDistCalculable (Maybe Bool) where
   integerDist Nothing a = 0
   integerDist a Nothing = 0 
   integerDist a b    
       | a == b = 1
       | a /= b = 0    

instance IntDistCalculable (Maybe Integer) where
   integerDist Nothing a = 0
   integerDist a Nothing = 0  
   integerDist a b 
       | a >= b = fromJust a - fromJust b 
       | a < b = fromJust b - fromJust a   

instance IntDistCalculable (Maybe Double) where
   integerDist Nothing a = 0
   integerDist a Nothing = 0  
   integerDist a b 
      | a >= b = round ( fromJust a - fromJust b )
      | a < b = round ( fromJust b - fromJust a )
